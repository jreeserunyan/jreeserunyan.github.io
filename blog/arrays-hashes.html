I want to be completely honest, so, I am not perfectly clear on the deeper concepts involved with hashes and arrays. But here are some disjointed thoughts, to be be refined later. Sorry for the lack of polish and significant content.

Arrays are containers of elements organized from first to last, accessed by indices starting at 0. If you are using brackets, it will look something like this: num_array = [1, 2, 3]. The left bracket starts or opens the array. Each element is separated by commas. The right bracket closes the array. They can contain any collection of objects, like strings, numbers, booleans and even other arrays. You can also do it with just `array = Array.new`.

  • Must capitalize Array (or Hash).
  • The above example is a literal constructor.
  • They can also be made with the .new method.
  • Hashes are array structures that don’t need numbers as indices. Instead, they can be looked up with the key in their key-value pair.
    • Use arrays for ordered lists. Use hashes for unordered lists.

    Each key in an a hash must be unique. It doesn’t matter what order they are in.

    Like arrays, hashes can be made with literal constructors and with the .new method. With the .each iterator, the variable between | | is a placeholder for each element of the object that the method .each is being used on.
