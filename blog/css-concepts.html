Positions, please!

CSS Concepts: What are the differences between relative, absolute, fixed, and static positioning?

Sept. 07, 2015

When setting up webpage with CSS, tiny differences can make huge changes. Positioning, display, floats and space are all important to the importance your site. The differences between the positioning values can be dramatic.
First, there is the default setting, which is static. There’s nothing really special about, the image will go with the normal flow of the page. It cannot be moved with properties like top, right, bottom and left.
The relative value moves an element in relation to its static position. When you use the position: relative; ruleset, you use additional properties, like top, right, bottom and left to direct the object where you want it to be.For example: top: 5em;.
The fixed property is a pretty interesting one. I would say it should be used sparingly. When you use the fixed position property, an element stays in the same place on the screen, regardless of scrolling and resizing. If you put it in upper right corner of you page, it will always be there!
The last positioning value is absolute. This one is a little tricky. Instead of being linked in relation to the page or the browser, it’s linked to the nearest positioned object. (A positioned object is just any object that isn’t static.) What this really means is that it can be relative to other relative objects on the screen, or to fixed objects. It’s a pretty useful thing, but it takes a little work to get used to it. I recommend just putting in some different values and watching the way the objects move in relation to each other.
