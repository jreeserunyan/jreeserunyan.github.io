Some bits of ruby

adapted from the poignant guide

arrays - a list surrounded by SQUARE BRACKETS and separated by commas.
[ ]
[1, 2, 3] is an array of numbers.
['coat', 'mittens', 'snowboard'] is an array of strings. Think of it as a caterpillar stalled into your code. The square brackets are the staples which keep the caterpillar from moving, so you know where it starts and where it ends. The commas are legs wriggling between the section of the body. It’s a collection of things in a specific order.

blocks - any code surrounded by curl braces is a block. With blocks you can group a set of instruction together so that they can be passed around your program.Ha curly braces give the appearance of crab pincers that have snatched the code and are holding it together. When you see pincers, remember that the code inside has been pressed into a single unit. You can also trade the curl braces for the keywords do and end for longer blocks.

block arguments - set of variables surrounded by pipe characters and separated with commas. Block arguments are used at the beginning of a block.
{ |x,y| x + y }
Above, |x,y| are arguments, after the arguments, there is a bit of code, the expression x + y adds the two arguments together. The pipe is a tunnel that the variables slide down. They pass through the tunnel and into the block, in this case x + y .

constants - capitalized, consisting of letters, digits and underscores. The proper nouns of ruby. The Empire State Building. You can’t just move The Empire State Building. It is what it is. They refer to very specific things that don’t change. Once a constant is set, Ruby will complain if you try to change a constant. Time, Array or Bunny_Lake_is_Missing are examples.

hashes - a dictionary surrounded by curly braces. Here our curly braces are are open books with creases in the center, the pen and close our dictionary. Easy to read through. Dictionaries match words to definitions. Ruby does so with arrows made for an equals sign followed by greater than sign => { 'name' => 'Peter', 'profession' => 'lion tamer', 'great love' => 'flannel' }
Imagine our dictionary has a definition on each of its pages. The commas represent the corner of each page, which we turn to see the next definition. And on each page: a word followed by an arrow pointing to the definition.

keywords - Ruby has a number of built-in words, imbued with meaning. These words cannot be used as variables or changed to suit your purposes. Some of these we’ve already discussed. They are in the safe house, my friend. You touch these and you’ll be served an official syntax error.

alias and BEGIN begin break case class def defined do else elsif END end ensure false for if in module next nil not or redo rescue retry return self super then true undef unless until when while yield

methods - If variables and constant are nouns, then methods are verbs of Ruby. Methods are attached to the end of variables and constant with a dot. The can have exclamation and question marks.
front_door.open
In the above, OPEN is the method. It’s the action. They can be chained together. front_door.open.close The door has now been instructed to open and then immediately close. front_door.is_open? The computer was instructed to test the door and see if it’s open.

class methods OR Instance methods - attached after variables and constant, using a double colon :: instead of a dot. Door::new( :oak ) Used to create things, here, make a new door out of oak.

kernel methods - Some methods, like print and exit are KERNEL methods, are so common, they don’t need a dot. They actually are class methods.

method arguments - A method may need more info to perform it’s action. If we want to paint the door, we should provide a color as well.
front_door.paint( 3, :red )

The above method and argument pains the front door 3 coats of red. Like an inner tube being pulled by a boat, the parentheses form wet round edges. Commas between arguments, but don’t leave a comma hanging at the end. Many methods can be chained.

front_door.paint( 3, :red ).dry( 30 ).close

The above paints the front door 3 coats of red paint, dries for 30 min and closes the door. You can put parentheses after the last method, even though it has no arguments, if you like. But they are usually dropped because it’s just dragging along something you don’t need.

numbers - basic integer, a series of digits, can begin with either a + or - . No commas allowed, but you want something for visual clarity, use underscores. (Floats consist of numbers with a decimal place or scientific notation.)

operators - You’ll use the following list of operators to do math in Ruby or to compare things. Scan over the list, recognize a few. You know, addition + and subtraction - and so on.
** ! ~ * / % + - & << >> | ^ > >= < <= <=> || != =~ !~ && += -= == === .. ... not and or

ranges - two values surround by parentheses, separated by two or three dots. (1..3)range, representing the numbers 1 through 3. ('a'..'z') range, representing a lowercase alphabet. (0...5) represents the numbers 0 through 4. Think of it as an accordion which has been squeezed down for carrying. The parentheses are the handles on the sides of a smaller, handheld accordion. The dots are the chain, keeping the folds tightly closed. Normally, only two dots are used. If a third dot is used, the last value in the range is excluded. (When you see that third dot, imagine opening the accordion slightly. Just enough to let one note from its chamber. The note is that end value. We’ll let the sky eat it.)

regular expressions - (regexp) set of characters surrounded by slashes. /ruby/, /[0-9]+/ and /^\d{3}-\d{3}-\d{4}/ are examples. Regular expressions are used to find words or patterns in text. The slashes on each side of the expression are pins. Imagine if you had a little word with pins on both side and you held it over a book. You pass the word over the book and when it gets near a matching word, it starts blinking. You pin the regular expression onto the book, right over the match and it glows with the letters of the matching word. Oh, and when you poke the pins into the book, the paper sneezes, reg-exp! Regular expressions are much faster than passing your hand over pages of a book. Ruby can use a regular expression to search volumes of books very quickly.

strings - any characters, letters, digit or punctuation surrounded by single or double quotes.

symbols - lowercase, consisting of letters, digits and underscores, but always beginning with a colon. Symbols function like lightweight strings. They are used in situations where you need a string but you won’t be printing it to the screen. :a. A symbol is “easy on the computer’s stomach”.

variables - lowercase, consisting of letters, digits and underscores. Like a nickname for something you use frequently. The nouns of Ruby.

instance variables - variables that begin with an AT symbol. @ Think of @ as meaning ATTRIBUTE. used to define characteristics of a single object. You might give Ruby the width of the front_door by setting the @width variable inside that front_door.

class variables - begin with double AT symbols @@ give an attribute to many related objects in Ruby. If instance variables set attributes for a single front_door, class variables set attributes for everything that is a Door. ATTRIBUTE ALL. or a swarm of star wars AT-AT(all terrain armored transport) all commanded by ruby. You change a class variable and they all change.

global variables - variables that begin with a dollar sign. $ Global variables can be accessed anywhere in your program. The never go out of sight.

Most variables are rather temporary in nature. Some parts of your program are like little houses. You walk in and they have their own variables. In one house, you may have a dad that represents Archie, a traveling salesman and skeleton collector. In another house, dad could represent Peter, a lion tamer with a great love for flannel. Each house has its own meaning for dad. With global variables, you can be guaranteed that the variable is the same in every little house. The dollar sign is very appropriate. Every American home respects the value of the dollar. We’re crazy for the stuff. Try knocking on any door in America and hand them cash. I can guarantee you won’t get the same reaction if you knock on a door and offer Peter, a lion tamer with a great love for flannel.
